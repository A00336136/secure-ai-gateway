Quick Setup

Before testing, ensure the following:

  1. Set BASE_URL below (default: http://localhost:8080)
  2. Start the Spring Boot app: JWT_SECRET=SecureAIGateway2026TUSMidlandsA00336136JWTSigningKeyHS256 java -jar secure-ai-web/target/secure-ai-web-2.0.0.jar --spring.profiles.active=dev
  3. For full pipeline testing, start Docker Compose: docker-compose up -d

1.1 User Registration

POST /auth/register - Create a new user account with BCrypt password hashing (cost factor 12)

Expected: HTTP 200 with username, role, message

1.2 User Login & JWT Token

POST /auth/login - Authenticate and receive a JWT token (HMAC-SHA384, 1-hour expiry, JTI for replay prevention)

Expected: JWT token, tokenType=Bearer, expiresIn=3600, role=USER

1.3 JWT Token Details

Decode and inspect the JWT token payload (base64-decoded, not verified)

Login first to see JWT details.

1.4 Auth Health Check

GET /auth/health - Public endpoint, no authentication required

2.1 API Status Check

GET /api/status - Check Ollama connectivity, rate limit remaining, and model info (requires JWT)

2.2 AI Inference - /api/ask

POST /api/ask - Send a prompt through the full security pipeline: JWT Auth → Rate Limit → 3-Layer Guardrails → LLM → PII Redaction → Audit Log

Response Headers: X-Rate-Limit-Remaining, X-PII-Redacted, X-Duration-Ms, X-Guardrails-Status

2.3 PII Detection & Redaction Test

Test the 12-pattern PII redaction engine. Enter text with sensitive data to see redaction in action.

3.1 Authentication Security Tests

Verify that protected endpoints reject unauthenticated and malformed requests

3.2 Input Validation Tests

Test server-side validation: username length, password length, empty fields

3.3 Rate Limiting Test

Bucket4j token bucket: 100 requests per hour per user. Send rapid requests to observe rate limit headers.

3.4 CORS Security Test

Verify CORS headers are returned correctly for allowed origins

3.5 Security Headers Verification

Verify HSTS, CSP, X-Frame-Options, Referrer-Policy, X-Content-Type-Options headers

Admin Access Required

Admin endpoints require a JWT token with role=ADMIN. In development mode, you can register an admin user by modifying the H2 database directly via the H2 Console at /h2-console.

H2 Console: JDBC URL = jdbc:h2:mem:secureaidb, User = sa, Password = (empty)

SQL: UPDATE USERS SET ROLE = 'ADMIN' WHERE USERNAME = 'testuser';

5.1 Dashboard Statistics

GET /admin/dashboard - Total requests, PII detections, rate-limited count, requests in last 24 hours

5.2 Audit Logs

GET /admin/audit - Paginated audit trail of all API requests

5.3 PII Alert Logs

GET /admin/audit/pii-alerts - Requests where PII was detected

5.4 Reset Rate Limit

DELETE /admin/rate-limit/{username} - Reset a user's rate limit bucket
