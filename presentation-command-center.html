6 · Anticipated Q&A — with our rehearsed answers

Eleven questions we've been drilling in dry runs. Each one has a primary responder and a backup.

Why parallel guardrails instead of sequential? — Mabin · backup Absar

Sequential forces the worst guardrail's latency to dominate. With Mono.zip the effective latency is max(NeMo, LlamaGuard, Presidio), not the sum — that's the 160ms → 90ms win on slide 7. Fail-closed logic ensures any block still denies even though the verdicts arrive out of order.

What stops someone removing the gateway and calling Ollama directly? — Absar

Ollama is on an isolated Docker bridge with no host port published — only the gateway container can reach 11434. The host-level firewall (ufw rules in k8s/networkpolicy.yaml) blocks ingress to Ollama's port from everything except the gateway service account. Physical & access-management controls sit outside our threat model and we say so in the paper.

Novel prompts — Palo Alto showed 85% → 34% accuracy drops. How do you handle those? — Richard

Three things. (1) Defence in depth — even if one layer misses a novel pattern, the other two still see the prompt. (2) NeMo Colang 2.0 lets us author rules reactively — a newly discovered jailbreak is one DSL commit, hot-reloaded, no retraining. (3) Future work (slide 10) includes drift monitoring per NIST AI RMF, which is exactly the policy answer to evolving adversarial inputs.

How do you actually reach 100% JaCoCo? Most projects claim 80%. — Mabin

We hit 100% across all six JaCoCo metrics — instruction, branch, line, complexity, method, class — via 175 methods in 10 JUnit 5 classes. A few methods were refactored to become testable; we did not use @ExcludeFromCoverage. The hallucination check bug on slide 9 was found precisely because branch hunting forced us into paths we had otherwise left untested.

Is LlamaGuard 3 biased / over-triggering? — Sabi

Our 50-prompt safe corpus produced 0 false positives — so no, not on that distribution. We acknowledge the corpus is small. NIST AI RMF drift monitoring (future work) is the answer for long-term bias auditing, along with Garak evaluations that we already run nightly in the AI-red-team Jenkins stage.

Why not use a commercial guardrail and be done with it? — Absar

Three blockers. (1) GDPR Article 25 — cloud transit of prompts is regularly considered a lawful-basis risk for sensitive enterprise data. (2) EU AI Act high-risk system obligations require full traceability, which cloud APIs don't surface. (3) Our Table III shows the cost gap — 87–93% over 3 years at all scales. The paper is explicit that a hybrid deployment is valid; we chose on-prem to prove the harder case.

Redis as a single point of failure? — Mabin

Today it's a single Redis 7.2 with Appendix-only persistence and RDB snapshots. Our slide 10 future-work item "Redis cluster mode for HA distributed state" is exactly this — Redis cluster with 3 shards and 3 replicas, which we've benchmarked but didn't ship in this iteration.

How did you source the 50 safe prompts — convenience sample? — Sabi

We stratified across four realistic enterprise categories — legal queries, finance analysis, software engineering help, and medical literature summarisation — plus a control set of pure small talk. The exact prompts are in karate-tests/features/safe-prompts.feature. We know 50 is a floor — we flag it as a limitation in the paper.

What's the failure mode if NeMo itself crashes? — Absar

Fail-closed. A Mono error from any layer is treated as a block — the request is denied and logged with blocked_reason: guardrail_unavailable. The panel can see that in console 3 — the decision engine does not peek at the response path. The Kubernetes liveness probe then restarts the container.

Reproducibility — can someone outside the team replicate Table I? — Richard

Yes. The public GitHub repo contains docker-compose.yml, the attack corpora under karate-tests/, and a single make evaluate target that regenerates every table. We committed the seeds for LlamaGuard's sampler so the 100% detection is deterministic.

Can your framework be used to audit existing LLM applications? — Mabin

Yes — deploy the gateway as a reverse proxy in front of an existing LLM endpoint. Upstream apps keep using OpenAI-style API shape; the guardrails sit transparently in front. We've proved this with a minimal Ollama target but the SPI is swappable.
