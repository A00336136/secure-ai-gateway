3. Sequence Diagram - POST /api/ask Request Flow

Runtime Behaviour
Figure 3. Sequence diagram for the POST /api/ask request flow. After JWT authentication and rate limiting, the GuardrailsOrchestrator executes three safety layers in parallel using Project Reactor's Mono.zip(): NeMo policy rules (~30ms), LlamaGuard ML classification (~60ms), and Presidio PII detection (~40ms). Combined guardrail latency is approximately 90ms. Upon passing all three layers, the prompt is forwarded to Ollama for Llama 3.2 inference (~1.5s), yielding a total end-to-end latency of approximately 1.6 seconds.

4. CI/CD Pipeline Diagram

15 Stages - Jenkins Declarative Pipeline
Figure 4. CI/CD pipeline with 15 stages implemented as a Jenkins Declarative Pipeline. The pipeline enforces a fail-fast strategy: quality gate failures (SonarQube) and critical vulnerability detections (Trivy, OWASP ZAP) halt the pipeline immediately. Total execution time is approximately 8-10 minutes for a complete build-test-security-deploy cycle.

5. 3-Layer Defence-in-Depth Diagram

Guardrails Architecture
Figure 5. Three-layer defence-in-depth architecture for LLM input validation. Layer 1 (NeMo Guardrails) applies policy-based rules using Colang DSL for jailbreak and topic safety. Layer 2 (LlamaGuard) performs ML-based classification against Meta's S1-S12 safety taxonomy. Layer 3 (Presidio) detects PII across 50+ entity types. All three layers execute in parallel via Project Reactor's Mono.zip(), achieving a combined latency of approximately 90ms. All layers must pass for the prompt to reach the LLM.
