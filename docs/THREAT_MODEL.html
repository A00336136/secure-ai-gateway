SecureAI Gateway — Systematic Threat Identification, Risk Assessment, and Mitigation Mapping
Research Paper Supplement • March 2026

1. STRIDE Threat Analysis

Each STRIDE category is mapped to specific threats against the SecureAI Gateway architecture, with corresponding mitigations linked to implementation artifacts.

STRIDE Category Threat Description Risk Mitigation Implementation Evidence
S — Spoofing Attacker impersonates a legitimate user or service to access the gateway API High JWT authentication with HMAC-SHA256 signing, token expiry enforcement, unique jti claim per token JwtTokenProvider.java, SecurityConfig.java, application.yml (jwt.secret, jwt.expiration)
T — Tampering Malicious modification of request payloads to bypass safety checks or inject harmful content High Input validation via @Size, @NotBlank Bean Validation annotations; Content Security Policy headers; CSRF protection ChatRequest.java (validation annotations), SecurityConfig.java (CSP headers), CsrfConfig.java
R — Repudiation User denies having made a specific request or received a particular response Medium Comprehensive audit logging via AuditLog JPA entity with timestamps, user ID tracking, request/response hashing AuditLog.java (entity), AuditService.java, AuditLogRepository.java
I — Information Disclosure PII leakage through LLM responses; stack traces exposing internals; sensitive data in logs Critical Presidio PII detection/redaction (50+ entity types); production error handling suppresses stack traces; structured logging without sensitive fields PresidioService.java, GlobalExceptionHandler.java, application-prod.yml
D — Denial of Service Resource exhaustion through excessive API requests or crafted large payloads High Token bucket rate limiter (100 req/hr per user); Resilience4j circuit breaker; request size limits via @Size annotation RateLimiterService.java, Resilience4jConfig.java, ChatRequest.java
E — Elevation of Privilege Non-admin user accessing admin-only endpoints; horizontal privilege escalation between users Critical RBAC with ADMIN/USER roles; SecurityFilterChain with endpoint-level authorization; method-level @PreAuthorize SecurityConfig.java, User.java (Role enum), AdminController.java

Risk Level Legend

  • Critical — Immediate remediation required
  • High — Remediation within current sprint
  • Medium — Scheduled remediation
  • Low — Accept or monitor

Identifiers

  • S = Spoofing, T = Tampering, R = Repudiation
  • I = Info Disclosure, D = Denial of Service, E = Elevation
  • LLM01–10 = OWASP LLM Top 10 (see Section 4)

4. OWASP LLM Top 10 Mapping

Systematic mapping of each OWASP LLM Top 10 (2025) risk to SecureAI Gateway mitigations with implementation references.

ID OWASP LLM Risk Applicability Mitigation Strategy Implementation Reference
LLM01 Prompt Injection Critical NeMo Guardrails with Colang rules detect and block jailbreak attempts, DAN-style prompts, and role-play exploitation vectors config/rails/jailbreak.co, NemoGuardrailsService.java
LLM02 Insecure Output Handling High Output sanitization layer; X-Content-Type-Options: nosniff; Content-Security-Policy headers prevent script execution from LLM output SecurityConfig.java (headers), ChatService.java (output processing)
LLM03 Training Data Poisoning Low (N/A) Not applicable — system uses pre-trained Ollama models without fine-tuning or training data ingestion docker-compose.yml (Ollama service), OllamaService.java
LLM04 Model Denial of Service High Token bucket rate limiter enforces 100 req/hr per user; Resilience4j circuit breaker prevents cascade failures to Ollama backend RateLimiterService.java, Resilience4jConfig.java
LLM05 Supply Chain Vulnerabilities High Trivy container image scanning; OWASP Dependency-Check for Java dependencies; SpotBugs static analysis in 15-stage Jenkins pipeline Jenkinsfile (stages: Trivy Scan, OWASP Check, SpotBugs), pom.xml (plugin configs)
LLM06 Sensitive Information Disclosure Critical Microsoft Presidio integration (Layer 3) detects 50+ PII entity types in both input and output; audit alerts triggered on detection PresidioService.java, SafetyOrchestrator.java, AuditService.java
LLM07 Insecure Plugin Design Low (N/A) Not applicable — SecureAI Gateway is a closed system with no plugin architecture; all integrations are compile-time dependencies pom.xml (dependency manifest)
LLM08 Excessive Agency Medium ReAct agent implementation bounded to maximum 5 iterations; no autonomous tool execution; all actions require explicit user prompts AgentService.java (MAX_ITERATIONS=5), AgentController.java
LLM09 Overreliance Medium Guardrails disclaimer headers appended to responses; confidence scoring on safety classifications; user-facing disclaimers SafetyOrchestrator.java (confidence thresholds), ChatController.java (response headers)
LLM10 Model Theft Medium Self-hosted Ollama with no external API exposure; model weights stored locally; no model serialization endpoints docker-compose.yml (Ollama network isolation), SecurityConfig.java (endpoint restrictions)
