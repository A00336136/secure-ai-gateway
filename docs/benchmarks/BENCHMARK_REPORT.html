Key Observations

  1. LLM inference dominates latency: Ollama (Llama 3.1 8B Q4_K_M) accounts for 87% of the /api/ask response time at p50. Guardrails overhead is only 5.5%.
  2. Parallel guardrails yield 2.9x speedup: Running Presidio PII detection and NeMo content safety concurrently via CompletableFuture reduces pre-processing overhead from 83ms to 45ms (bounded by the slower component).
  3. BCrypt cost factor is the auth bottleneck: At the default strength of 10, login/register operations consistently take 110-130ms. This is intentional for security but could be tuned for local development.
  4. ReAct agent incurs 2.2x overhead: Multi-step reasoning requires 2-3 sequential LLM calls, making the ReAct path approximately 2.2x slower than single-shot inference at p50.
  5. Rate limiter and RBAC are negligible: Token bucket checks and role-based access control add under 25ms combined, confirming in-memory implementations are efficient.
  6. Throughput saturates at C=10: The /api/ask endpoint reaches only 3.1 req/s at C=10 due to Ollama serializing GPU inference. Health endpoints scale linearly to 840 req/s.
